## Getting Started

### Prerequisites
- **Node.js**: v18+ (recommended: v20)
- **Bun**: Latest version (package manager)
- **Git**: For version control
- **VS Code**: Recommended IDE with extensions:
  - ESLint
  - Prettier
  - Tailwind CSS IntelliSense
  - TypeScript

### Initial Setup

1. **Clone the repository**:
   ```bash
   git clone <repository-url>
   cd kartels-io
   ```

2. **Install dependencies**:
   ```bash
   bun install
   ```

3. **Start development server**:
   ```bash
   bun run dev
   ```

4. **Open in browser**:
   ```
   http://localhost:8080
   ```

### Environment Variables

The `.env` file is auto-generated by Lovable Cloud. Key variables:

```env
VITE_SUPABASE_URL=https://crtqzmaiuwipkzfszkmb.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=<your-key>
VITE_SUPABASE_PROJECT_ID=crtqzmaiuwipkzfszkmb
```

**Never commit `.env` to version control!**

## Project Structure

See [ARCHITECTURE.md](./ARCHITECTURE.md) for detailed structure.

### Key Directories

```
src/
├── components/       # Reusable UI components
├── pages/            # Route-level components
├── services/         # API/business logic
├── lib/              # Utilities, hooks, constants
├── contexts/         # React Context providers
├── locales/          # Translation files
└── integrations/     # Supabase client (auto-generated)
```

### File Naming Conventions

- **Components**: PascalCase (e.g., `UserMenu.tsx`)
- **Services**: camelCase with `.service.ts` suffix (e.g., `auth.service.ts`)
- **Utilities**: camelCase (e.g., `date.ts`, `validation.ts`)
- **Hooks**: camelCase with `use` prefix (e.g., `useCartel.ts`)
- **Types**: PascalCase for interfaces (e.g., `UserProfile`)

## Development Workflow

### Branch Strategy

- `main` - Production-ready code
- `develop` - Integration branch
- Feature branches: `feature/your-feature-name`
- Bug fixes: `fix/bug-description`

### Commit Messages

Follow conventional commits:

```
feat: add user profile editing
fix: resolve navigation bug on mobile
docs: update README with setup instructions
refactor: improve data fetching logic
style: format code with Prettier
test: add tests for auth service
```

- [ ] Code follows style guidelines

- [ ] Responsive design tested on mobile/tablet/desktop

## Adding New Features

### Step-by-Step Guide

#### 1. Plan the Feature

- Define requirements
- Identify affected modules
- Design data model (if database changes needed)
- Create mockups/wireframes

#### 2. Database Changes (if needed)

If your feature requires new tables or columns:

```sql
-- Example: Add a new table
CREATE TABLE public.my_new_table (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  cartel_id UUID REFERENCES public.cartels(id) NOT NULL,
  user_id UUID REFERENCES auth.users(id) NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Enable RLS
ALTER TABLE public.my_new_table ENABLE ROW LEVEL SECURITY;

-- Add policies
CREATE POLICY "Users can view own data"
  ON public.my_new_table
  FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own data"
  ON public.my_new_table
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);
```

Run migration via Lovable Cloud interface or Supabase CLI.

#### 3. Create Service Layer

**File**: `src/services/my-feature.service.ts`

```typescript
/**
 * My Feature Service
 * 
 * Handles data operations for MyFeature.
 * 
 * @module services/my-feature
 */

import { supabase } from '@/integrations/supabase/client';

export interface MyFeatureData {
  id: string;
  content: string;
  created_at: string;
}

/**
 * Fetches all items for a cartel
 * 
 * @param cartelId - Cartel UUID
 * @returns Promise with array of items
 */
export async function getItems(cartelId: string): Promise<MyFeatureData[]> {
  const { data, error } = await supabase
    .from('my_new_table')
    .select('*')
    .eq('cartel_id', cartelId)
    .order('created_at', { ascending: false });

  if (error) {
    console.error('Error fetching items:', error);
    return [];
  }

  return data || [];
}

/**
 * Creates a new item
 * 
 * @param item - Item data to create
 * @returns Promise with created item or null
 */
export async function createItem(item: Omit<MyFeatureData, 'id' | 'created_at'>): Promise<MyFeatureData | null> {
  const { data, error } = await supabase
    .from('my_new_table')
    .insert(item)
    .select()
    .single();

  if (error) {
    console.error('Error creating item:', error);
    return null;
  }

  return data;
}
```

#### 4. Create Component

**File**: `src/components/dashboard/MyFeature.tsx`

```typescript
/**
 * MyFeature Component
 * 
 * Displays and manages MyFeature data for a cartel.
 * Users can view, create, and edit items.
 */

import { useState, useEffect } from 'react';
import { useAuth } from '@/contexts/AuthContext';
import { useCartel } from '@/lib/hooks/useCartel';
import { getItems, createItem, type MyFeatureData } from '@/services/my-feature.service';
import { Card, CardHeader, CardTitle, CardContent } from '@/components/ui/card';
import { Button } from '@/components/ui/button';
import { toast } from 'sonner';

export const MyFeature = () => {
  const { user } = useAuth();
  const { activeCartel } = useCartel();
  const [items, setItems] = useState<MyFeatureData[]>([]);
  const [loading, setLoading] = useState(true);

  // Fetch data on mount
  useEffect(() => {
    if (activeCartel) {
      fetchItems();
    }
  }, [activeCartel]);

  const fetchItems = async () => {
    if (!activeCartel) return;
    setLoading(true);
    const data = await getItems(activeCartel.id);
    setItems(data);
    setLoading(false);
  };

  const handleCreate = async () => {
    if (!activeCartel || !user) return;
    
    const newItem = {
      cartel_id: activeCartel.id,
      user_id: user.id,
      content: 'New item',
    };

    const created = await createItem(newItem);
    if (created) {
      setItems([created, ...items]);
      toast.success('Item created successfully!');
    } else {
      toast.error('Failed to create item');
    }
  };

  if (loading) {
    return <div>Loading...</div>;
  }

  return (
    <Card>
      <CardHeader>
        <CardTitle>My Feature</CardTitle>
      </CardHeader>
      <CardContent>
        <Button onClick={handleCreate}>Create Item</Button>
        <div className="mt-4 space-y-2">
          {items.map((item) => (
            <div key={item.id} className="p-2 border rounded">
              {item.content}
            </div>
          ))}
        </div>
      </CardContent>
    </Card>
  );
};
```

## Working with Supabase

### Querying Data

```typescript
// SELECT
const { data, error } = await supabase
  .from('table_name')
  .select('*')
  .eq('column', 'value');

// INSERT
const { data, error } = await supabase
  .from('table_name')
  .insert({ column: 'value' })
  .select()
  .single();

// UPDATE
const { data, error } = await supabase
  .from('table_name')
  .update({ column: 'new_value' })
  .eq('id', 'some-id')
  .select()
  .single();

// DELETE
const { error } = await supabase
  .from('table_name')
  .delete()
  .eq('id', 'some-id');
```

### Real-time Subscriptions

```typescript
useEffect(() => {
  const channel = supabase
    .channel('my-channel')
    .on(
      'postgres_changes',
      {
        event: '*', // 'INSERT', 'UPDATE', 'DELETE', or '*'
        schema: 'public',
        table: 'table_name',
        filter: 'cartel_id=eq.123', // Optional filter
      },
      (payload) => {
        console.log('Change received!', payload);
        // Update local state
      }
    )
    .subscribe();

  return () => {
    supabase.removeChannel(channel);
  };
}, []);
```

### Authentication

Use the `AuthContext`:
```typescript
const { user, session, loading, signOut } = useAuth();
```

Get current user ID:
```typescript
const userId = user?.id;
```

## Internationalization (i18n)

### Adding Translations

1. Add keys to `src/locales/en.json` and `src/locales/fr.json`
2. Use the `useTranslation` hook:

```typescript
const { t } = useTranslation();

return (
  <div>
    <h1>{t('dashboard.welcome')}</h1>
    <p>{t('dashboard.description', { name: userName })}</p>
  </div>
);
```

### Translation File Structure

```json
{
  "nav": { ... },
  "dashboard": {
    "welcome": "Welcome",
    "sections": {
      "overview": "Overview",
      "calendar": "Calendar"
    }
  }
}
```

### Pluralization

```json
{
  "items": "{{count}} item",
  "items_plural": "{{count}} items"
}
```

Usage:
```typescript
{t('items', { count: itemCount })}
```

## Styling Guidelines

### Tailwind CSS

Use Tailwind utility classes. **Never** use custom CSS unless absolutely necessary.

```tsx
<div className="flex items-center justify-between p-4 bg-background border border-border rounded-lg">
  <h2 className="text-xl font-bold text-foreground">Title</h2>
  <Button className="bg-primary text-primary-foreground">Action</Button>
</div>
```

### Design Tokens

Use semantic tokens from `index.css`:

- `bg-background` / `bg-foreground`
- `text-foreground` / `text-muted-foreground`
- `border-border`
- `bg-primary` / `text-primary-foreground`
- `bg-accent` / `text-accent-foreground`
- `bg-success` / `bg-destructive`

**Never** use raw colors like `bg-white`, `text-black`, etc.

### Responsive Design

```tsx
<div className="flex flex-col md:flex-row gap-4">
  <div className="w-full md:w-1/2">Left</div>
  <div className="w-full md:w-1/2">Right</div>
</div>
```

Breakpoints:
- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px
- `2xl`: 1536px

## Common Patterns

### Custom Hook Pattern

```typescript
export function useMyData(cartelId: string) {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function fetch() {
      setLoading(true);
      const result = await getMyData(cartelId);
      setData(result);
      setLoading(false);
    }
    fetch();
  }, [cartelId]);

  return { data, loading, refetch: fetch };
}
```

### Error Handling Pattern

```typescript
try {
  const result = await someOperation();
  toast.success('Operation successful!');
  return result;
} catch (error) {
  console.error('Error in someOperation:', error);
  toast.error('Operation failed. Please try again.');
  return null;
}
```

### Loading State Pattern

```typescript
if (loading) {
  return (
    <div className="flex items-center justify-center p-12">
      <div className="animate-spin rounded-full h-12 w-12 border-b-2 border-primary"></div>
    </div>
  );
}
```

## Troubleshooting

### Common Issues

#### Issue: TypeScript errors after Supabase schema change
**Solution**: Types are auto-generated. Restart dev server:
```bash
bun run dev
```

#### Issue: Component not updating after data change
**Solution**: Check if you're properly updating state. Use React Query for automatic refetching.

#### Issue: i18n key not found
**Solution**: Check spelling in `en.json` and `fr.json`. Ensure keys match exactly.

#### Issue: Build fails with module not found
**Solution**: Check import paths. Use `@/` alias for src folder:
```typescript
import { MyComponent } from '@/components/MyComponent';
```

### Debug Tips

1. **React DevTools**: Install browser extension for component inspection
2. **Supabase Logs**: Check Lovable Cloud interface for database errors
3. **Network Tab**: Inspect API calls and responses
4. **Console**: Check for errors and warnings
